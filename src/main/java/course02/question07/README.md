## 1. 선택 정렬을 사용한 이유

이번 문제에서는 동물 이름을 단순히 가나다순으로만 정렬하는 것이 아니라, 두 가지 기준을 함께 사용해야 한다.

1. 관찰 빈도수가 높은 동물이 먼저 온다.
2. 관찰 빈도수가 같으면 동물 이름을 가나다순으로 정렬한다.

이처럼 정렬 기준이 두 개이므로, 정렬 과정에서 어떤 동물이 앞에 와야 하는지 직접 비교해야 한다.

구현에서는 선택 정렬(Selection Sort)을 사용했다. 선택 정렬은 현재 위치에 들어갈 가장 적절한 값을 남은 구간에서 찾고, 현재 위치와 교환하는 방식이다.

선택 정렬을 고른 이유는 다음과 같다.

- 구현 구조가 단순하다.
- 현재 위치에 올 동물을 직접 고르는 방식이라 정렬 기준을 이해하기 쉽다.
- 빈도수 내림차순과 이름 가나다순 조건을 함께 적용하기 쉽다.

코드에서는 다음 조건으로 어떤 동물이 먼저 와야 하는지 판단한다.

```java
if (firstObservationCount != secondObservationCount) {
    return firstObservationCount > secondObservationCount;
}

return firstAnimalName.compareTo(secondAnimalName) < 0;
```

먼저 관찰 빈도수를 비교한다. 빈도수가 다르면 더 많이 관찰된 동물이 앞에 온다.

빈도수가 같으면 `compareTo()`로 동물 이름을 비교한다. 이때 결과가 음수이면 앞 이름이 뒤 이름보다 사전순으로 앞에 있다는 뜻이므로, 앞에 배치한다.

## 2. 선택 정렬의 작동 방식

예를 들어 고유 동물 이름과 빈도수가 다음과 같다고 하자.

```text
사자: 4
얼룩말: 2
악어: 3
표범: 2
늑대: 3
```

정렬 기준은 빈도수 내림차순이다. 따라서 가장 먼저 와야 하는 동물은 빈도수가 `4`인 `사자`이다.

그 다음 남은 동물 중 빈도수가 높은 동물은 `악어`와 `늑대`이다.

```text
악어: 3
늑대: 3
```

두 동물의 빈도수가 같으므로 이름을 가나다순으로 비교한다. `늑대`가 `악어`보다 가나다순으로 앞에 오므로 `늑대`가 먼저 출력된다.

최종 결과는 다음과 같다.

```text
사자, 늑대, 악어, 얼룩말, 표범
```

선택 정렬은 이 과정을 각 위치마다 반복한다.

```text
1번째 위치에 올 동물을 찾는다.
2번째 위치에 올 동물을 찾는다.
3번째 위치에 올 동물을 찾는다.
...
```

즉 매번 남은 동물 중에서 가장 먼저 출력되어야 하는 동물을 선택한다.

## 3. 선택 정렬의 시간 복잡도

선택 정렬은 배열이 어떤 상태이든 현재 위치 뒤쪽의 모든 값을 확인하면서 가장 적절한 값을 찾는다.

따라서 최악, 평균, 최선의 경우 모두 시간 복잡도가 같다.

| 경우 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| 최악의 경우 | `O(n^2)` | 모든 위치에서 남은 동물들을 비교해야 한다. |
| 평균적인 경우 | `O(n^2)` | 일반적인 입력에서도 중첩 반복문으로 비교한다. |
| 최선의 경우 | `O(n^2)` | 이미 정렬되어 있어도 남은 구간을 계속 확인한다. |

여기서 `n`은 고유한 동물 이름의 개수이다.

예를 들어 고유 동물이 5종이면 첫 번째 위치를 정하기 위해 4번 비교하고, 두 번째 위치를 정하기 위해 3번 비교한다.

```text
4 + 3 + 2 + 1
```

고유 동물 개수가 `n`개이면 비교 횟수는 대략 다음과 같다.

```text
(n - 1) + (n - 2) + ... + 1
```

Big O 표기법에서는 상수와 낮은 차수 항을 생략하므로 시간 복잡도는 다음과 같다.

```text
O(n^2)
```

## 4. 더 나은 방법

선택 정렬은 이해하기 쉽지만, 데이터가 많아지면 비효율적이다.

더 나은 방법으로는 병합 정렬(Merge Sort)이나 퀵 정렬(Quick Sort)을 사용할 수 있다.

| 정렬 방식 | 평균 시간 복잡도 | 특징 |
| :--- | :--- | :--- |
| 선택 정렬 | `O(n^2)` | 구현이 단순하지만 큰 데이터에 느리다. |
| 퀵 정렬 | `O(nlogn)` | 평균적으로 빠르지만 피벗 선택에 따라 최악의 경우 `O(n^2)`이 될 수 있다. |
| 병합 정렬 | `O(nlogn)` | 항상 안정적인 `O(nlogn)` 성능을 가지지만 추가 배열이 필요하다. |

이번 문제의 보너스 과제에서는 병합 정렬을 이용해 같은 기능을 구현하고, 선택 정렬과 시간 복잡도를 비교할 수 있다.

병합 정렬을 사용하면 고유 동물 이름이 많아질수록 선택 정렬보다 효율적이다.

## 5. compareTo() 메서드의 작동 원리

`compareTo()`는 문자열을 사전순으로 비교하는 메서드이다.

사용 형식은 다음과 같다.

```java
firstAnimalName.compareTo(secondAnimalName)
```

결과는 정수로 나온다.

| 결과 | 의미 |
| :--- | :--- |
| 음수 | 앞 문자열이 뒤 문자열보다 사전순으로 앞에 있다. |
| 0 | 두 문자열이 같다. |
| 양수 | 앞 문자열이 뒤 문자열보다 사전순으로 뒤에 있다. |

예를 들어 다음과 같이 비교할 수 있다.

```java
"늑대".compareTo("악어")
```

`늑대`는 `악어`보다 가나다순으로 앞에 오므로 결과는 음수이다.

```java
"표범".compareTo("얼룩말")
```

`표범`은 `얼룩말`보다 가나다순으로 뒤에 오므로 결과는 양수이다.

따라서 코드에서는 빈도수가 같은 경우 다음 조건을 사용한다.

```java
firstAnimalName.compareTo(secondAnimalName) < 0
```

이 조건이 참이면 `firstAnimalName`이 `secondAnimalName`보다 가나다순으로 앞에 있으므로 먼저 출력한다.

## 6. 전체 구현 흐름

프로그램의 전체 흐름은 다음과 같다.

1. Command Line Arguments로 입력 문자열을 받는다.
2. 문자열 형태의 배열에서 동물 이름을 파싱한다.
3. 숫자나 빈 값이 포함되어 있으면 잘못된 입력으로 처리한다.
4. 중복을 제거해 고유한 동물 이름 배열을 만든다.
5. 각 고유 동물 이름이 전체 입력에서 몇 번 등장했는지 계산한다.
6. 선택 정렬로 관찰 빈도수 내림차순 정렬을 수행한다.
7. 빈도수가 같으면 `compareTo()`를 사용해 가나다순으로 정렬한다.
8. 정렬된 동물 이름만 출력한다.

예시 입력:

```text
["사자", "얼룩말", "악어", "표범", "사자", "늑대", "악어", "늑대", "사자", "표범", "늑대", "악어", "얼룩말", "사자"]
```

빈도수:

```text
사자: 4
늑대: 3
악어: 3
얼룩말: 2
표범: 2
```

정렬 결과:

```text
사자, 늑대, 악어, 얼룩말, 표범
```
