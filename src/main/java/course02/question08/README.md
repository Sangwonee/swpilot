## 개념 확인 질문 정리

### 1. 큐(Queue)의 핵심 원리는 무엇이며 스택(Stack)과 어떤 차이가 있는가?

큐는 먼저 들어온 데이터를 먼저 꺼내는 자료구조이다. 이를 FIFO(First-In-First-Out), 즉 선입선출이라고 한다.

예를 들어 자원 요청이 `100 → 200 → 300` 순서로 들어오면, 큐에서는 같은 순서인 `100 → 200 → 300`으로 처리한다. 줄을 먼저 선 요청부터 처리하는 것과 같은 방식이다.

스택은 가장 나중에 들어온 데이터를 먼저 꺼내는 자료구조이다. 이를 LIFO(Last-In-First-Out), 즉 후입선출이라고 한다.

같은 값이 `100 → 200 → 300` 순서로 들어오면, 스택에서는 `300 → 200 → 100` 순서로 꺼낸다. 위로 쌓인 물건을 가장 위에서부터 꺼내는 것과 같은 방식이다.

| 자료구조 | 처리 원리 | 입력 순서 | 출력 순서 |
|---|---|---|---|
| 큐(Queue) | FIFO, 선입선출 | 100 → 200 → 300 | 100 → 200 → 300 |
| 스택(Stack) | LIFO, 후입선출 | 100 → 200 → 300 | 300 → 200 → 100 |

이번 문제에서는 식물의 자원 요청을 접수된 순서대로 처리해야 하므로 스택이 아니라 큐가 적합하다.

### 2. `dequeue`와 `peek` 메서드의 차이는 무엇인가?

두 메서드 모두 큐의 가장 앞에 있는 값을 대상으로 하지만, 값을 제거하는지가 다르다.

- `peek()`는 가장 앞의 값을 확인만 하고 큐에서 제거하지 않는다.
- `dequeue()`는 가장 앞의 값을 꺼내면서 큐에서 제거한다.

큐에 `[100, 200, 300]`이 들어 있다고 가정하면 다음과 같다.

```text
peek() 실행
반환값: 100
실행 후 큐: [100, 200, 300]

dequeue() 실행
반환값: 100
실행 후 큐: [200, 300]
```

따라서 요청값을 처리하기 전에 확인만 해야 할 때는 `peek()`를 사용하고, 요청 처리를 완료하여 큐에서 없애야 할 때는 `dequeue()`를 사용한다.

이번 구현에서는 `peek()`으로 제공할 자원의 양을 먼저 확인한 다음, `dequeue()`로 해당 요청을 큐에서 제거한다.
