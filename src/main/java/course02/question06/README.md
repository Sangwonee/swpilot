## 구현 정리

`RoadToBiodome06.java`는 두 개의 물 높이 배열을 입력받아 전체 평균값과 중앙값을 출력하도록 구현했다.

입력은 Command Line Arguments로 받는다. 여러 인자로 나뉘어 들어올 수 있으므로 `String.join(" ", args)`를 사용해 하나의 문자열로 합친 뒤 처리한다.

```java
String input = String.join(" ", args).trim();
```

입력이 없거나 공백만 있으면 잘못된 입력으로 보고 안내 메시지를 출력한다.

---

## 1. 두 배열 입력 파싱

입력 문자열에서 첫 번째 배열과 두 번째 배열의 시작과 끝 위치를 찾는다.

```java
int firstStart = input.indexOf('[');
int firstEnd = input.indexOf(']', firstStart + 1);
int secondStart = input.indexOf('[', firstEnd + 1);
int secondEnd = input.indexOf(']', secondStart + 1);
```

각 변수의 의미는 다음과 같다.

| 변수 | 의미 |
| :--- | :--- |
| `firstStart` | 첫 번째 배열의 `[` 위치 |
| `firstEnd` | 첫 번째 배열의 `]` 위치 |
| `secondStart` | 두 번째 배열의 `[` 위치 |
| `secondEnd` | 두 번째 배열의 `]` 위치 |

예를 들어 입력이 다음과 같다면:

```text
[2, 10, 30] [1, 9, 27, 40, 42]
```

첫 번째 배열 내부 문자열은 다음처럼 잘라낸다.

```java
input.substring(firstStart + 1, firstEnd)
```

결과:

```text
2, 10, 30
```

두 번째 배열 내부 문자열은 다음처럼 잘라낸다.

```java
input.substring(secondStart + 1, secondEnd)
```

결과:

```text
1, 9, 27, 40, 42
```

배열 앞, 배열 사이, 배열 뒤에 공백이 아닌 다른 문자가 있으면 잘못된 입력으로 처리한다. 또한 배열이 2개보다 많거나 괄호가 남아 있어도 잘못된 입력으로 처리한다.

---

## 2. 배열 내부 값 변환과 예외 처리

배열 내부 문자열은 콤마 `,`를 기준으로 나눈다.

```java
String[] values = trimmedArrayText.split(",", -1);
```

`-1`을 사용한 이유는 빈 값도 버리지 않고 검사하기 위해서이다.

예를 들어 다음 입력은 잘못된 입력이다.

```text
[1, 2,]
```

배열 내부 문자열은 다음과 같다.

```text
1, 2,
```

`split(",", -1)`을 사용하면 결과는 다음과 같다.

```text
["1", " 2", ""]
```

마지막 빈 문자열 `""`이 남기 때문에 숫자가 비어 있다는 것을 확인할 수 있다.

각 값은 앞뒤 공백을 제거한 뒤 정수로 변환한다.

```java
int waterHeight = Integer.parseInt(value);
```

`Integer.parseInt()`를 사용했기 때문에 숫자가 아니거나 `int` 범위를 벗어난 값은 바로 예외가 발생한다. 이후 물 높이가 허용 범위인 `0~100` 안에 있는지 검사한다.

```java
if (waterHeight < MIN_WATER_HEIGHT || waterHeight > MAX_WATER_HEIGHT) {
    throw new NumberFormatException();
}
```

잘못된 입력이 발견되면 `NumberFormatException`을 발생시키고, `main()`의 `catch`에서 안내 메시지를 출력한다.

## 3. 평균값 계산 방식

평균값은 두 배열의 모든 값을 더한 뒤 전체 개수로 나누어 구한다.

```java
public static double calculateMean(int[] firstArray, int[] secondArray) {
    long sum = 0;

    for (int i = 0; i < firstArray.length; i++) {
        sum += firstArray[i];
    }
    for (int i = 0; i < secondArray.length; i++) {
        sum += secondArray[i];
    }

    return (double) sum / (firstArray.length + secondArray.length);
}
```

두 배열의 모든 값을 한 번씩 확인하므로 평균값 계산의 시간 복잡도는 다음과 같다.

```text
O(n + m)
```

여기서 `n`은 첫 번째 배열의 길이, `m`은 두 번째 배열의 길이이다.

---

## 4. 중앙값 계산 방식

중앙값은 두 배열을 하나로 합쳐 정렬하지 않고 계산한다.

두 배열은 이미 오름차순으로 정렬되어 있다는 전제이므로, 이진 탐색을 사용해 두 배열을 왼쪽 구간과 오른쪽 구간으로 나눈다.

목표는 다음 조건을 만족하는 분할 위치를 찾는 것이다.

```text
왼쪽 구간의 모든 값 <= 오른쪽 구간의 모든 값
```

코드에서는 다음 조건으로 올바른 분할인지 확인한다.

```java
if (firstLeft <= secondRight && secondLeft <= firstRight)
```

올바른 분할을 찾으면 전체 데이터 개수에 따라 중앙값을 구한다.

전체 개수가 홀수이면 왼쪽 구간에서 가장 큰 값이 중앙값이다.

```java
return max(firstLeft, secondLeft);
```

전체 개수가 짝수이면 왼쪽 구간의 가장 큰 값과 오른쪽 구간의 가장 작은 값의 평균이 중앙값이다.

```java
return (max(firstLeft, secondLeft) + min(firstRight, secondRight)) / 2.0;
```

---

## 5. 중앙값 계산 예시

예를 들어 두 배열이 다음과 같이 정렬되어 있다고 하자.

```text
[2, 10, 30] [1, 9, 27, 40, 42]
```

두 배열을 실제로 합치지는 않지만, 이해를 위해 전체 정렬 상태를 보면 다음과 같다.

```text
[1, 2, 9, 10, 27, 30, 40, 42]
```

전체 개수는 `8`개로 짝수이다. 가운데 두 값은 `10`과 `27`이다.

```text
(10 + 27) / 2 = 18.5
```

따라서 중앙값은 `18.5`이다.

현재 구현은 이 값을 구하기 위해 배열을 새로 만들거나 정렬하지 않고, 분할 위치를 이진 탐색으로 찾아 계산한다.

---

## 6. 시간 복잡도

평균값 계산은 두 배열의 모든 값을 한 번씩 더해야 하므로 `O(n + m)`이다.

중앙값 계산은 더 짧은 배열을 기준으로 이진 탐색을 수행한다. 따라서 시간 복잡도는 다음과 같다.

```text
O(log(min(n, m)))
```

이는 `O(log(n + m))`보다 더 작거나 같은 범위이므로 요구사항의 중앙값 계산 시간 복잡도 조건을 만족한다.

배열을 하나로 합쳐 정렬하지 않기 때문에 중앙값 계산 과정에서 추가 정렬 비용은 발생하지 않는다.

## 7. 출력 형식

평균값과 중앙값은 소수점 첫째 자리까지 반올림해 출력한다.

```java
private static String formatOneDecimal(double value) {
    long scaled = (long) (value * 10 + 0.5);
    return (scaled / 10) + "." + (scaled % 10);
}
```

예를 들어 `20.125`는 다음 과정을 거쳐 `20.1`로 출력된다.

```text
20.125 * 10 = 201.25
201.25 + 0.5 = 201.75
(long) 201.75 = 201
201 / 10 = 20
201 % 10 = 1
```

최종 출력은 다음 형식을 사용한다.

```text
Mean : 20.1, Median : 18.5
```
