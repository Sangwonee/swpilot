## 1. 퀵 정렬 알고리즘의 작동 원리

퀵 정렬(Quick Sort)은 배열에서 기준값인 피벗(pivot)을 하나 정한 뒤, 피벗보다 작거나 같은 값은 왼쪽에 두고 피벗보다 큰 값은 오른쪽에 두는 방식으로 정렬하는 알고리즘이다.

피벗을 기준으로 배열을 나누는 과정을 분할(partition)이라고 한다. 분할이 끝나면 피벗은 정렬된 배열에서 자신의 최종 위치에 놓이게 된다.

작동 방식은 다음과 같다.

1. 배열에서 피벗을 하나 선택한다.
2. 피벗보다 작거나 같은 값은 피벗의 왼쪽으로 이동한다.
3. 피벗보다 큰 값은 피벗의 오른쪽으로 이동한다.
4. 피벗을 기준으로 나뉜 왼쪽 배열과 오른쪽 배열에 같은 과정을 반복한다.
5. 더 이상 나눌 수 없을 때 정렬이 완료된다.

예를 들어 다음 배열이 있다.

```text
[5, 3, 8, 4, 2]
```

현재 구현에서는 정렬 구간의 마지막 값을 피벗으로 사용한다. 따라서 첫 번째 피벗은 `2`이다.

### 1회 분할

피벗 `2`보다 작거나 같은 값이 없으므로 `2`는 배열의 맨 앞에 놓인다.

```text
[2, 3, 8, 4, 5]
```

이제 `2`는 최종 위치가 확정되었고, 오른쪽 구간 `[3, 8, 4, 5]`를 다시 정렬한다.

### 2회 분할

오른쪽 구간 `[3, 8, 4, 5]`의 피벗은 `5`이다.

`5`보다 작거나 같은 값인 `3`, `4`는 왼쪽에 놓이고, `8`은 오른쪽에 놓인다.

```text
[2, 3, 4, 5, 8]
```

이제 `5`의 위치도 확정된다.

### 3회 분할

남은 왼쪽 구간 `[3, 4]`의 피벗은 `4`이다.

`3`은 `4`보다 작으므로 왼쪽에 그대로 있고, `4`도 최종 위치가 확정된다.

```text
[2, 3, 4, 5, 8]
```

모든 구간의 정렬이 끝났으므로 최종 결과는 다음과 같다.

```text
[2, 3, 4, 5, 8]
```

코드에서는 다음 부분이 퀵 정렬을 수행한다.

```java
public static void quickSort(int[] numbers, int left, int right) {
    if (left >= right) {
        return;
    }

    int pivotIndex = partition(numbers, left, right);
    quickSort(numbers, left, pivotIndex - 1);
    quickSort(numbers, pivotIndex + 1, right);
}
```

- `partition()`은 피벗을 기준으로 배열을 왼쪽과 오른쪽으로 나눈다.
- `quickSort(numbers, left, pivotIndex - 1)`은 피벗 왼쪽 구간을 다시 정렬한다.
- `quickSort(numbers, pivotIndex + 1, right)`은 피벗 오른쪽 구간을 다시 정렬한다.
- `left >= right`가 되면 더 이상 정렬할 값이 없으므로 재귀 호출을 멈춘다.

## 2. 퀵 정렬의 시간 복잡도

퀵 정렬의 시간 복잡도는 피벗이 배열을 얼마나 균형 있게 나누는지에 따라 달라진다.

| 경우 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| 최악의 경우 | `O(n^2)` | 피벗이 계속 가장 작거나 큰 값이 되어 한쪽 구간만 길게 남는 경우이다. |
| 평균적인 경우 | `O(nlogn)` | 피벗이 배열을 비교적 균형 있게 나누는 일반적인 경우이다. |
| 최선의 경우 | `O(nlogn)` | 피벗이 매번 배열을 거의 절반씩 나누는 경우이다. |

퀵 정렬은 한 번 분할할 때 배열의 각 값을 피벗과 비교하므로 `O(n)`만큼 시간이 걸린다.

피벗이 배열을 절반씩 잘 나누면 분할 단계의 깊이는 대략 `logn`이 된다. 따라서 전체 시간 복잡도는 다음과 같다.

```text
O(nlogn)
```

하지만 피벗이 계속 가장 작거나 가장 큰 값으로 선택되면 배열이 거의 나누어지지 않는다.

예를 들어 이미 오름차순 정렬된 배열에서 마지막 값을 피벗으로 고르면, 피벗은 계속 가장 큰 값이 된다.

```text
[1, 2, 3, 4, 5]
```

이 경우 한 번 분할할 때마다 한쪽 구간에만 값이 몰리므로 재귀 호출 깊이가 `n`에 가까워진다. 그래서 최악의 경우 시간 복잡도는 다음과 같다.

```text
O(n^2)
```

## 3. 퀵 정렬과 선택 정렬의 차이

선택 정렬(Selection Sort)은 배열에서 가장 작은 값을 찾아 앞쪽부터 하나씩 확정하는 방식이다. 입력 배열이 어떤 상태이든 남은 값들을 계속 비교해야 하므로 최악, 평균, 최선 모두 `O(n^2)`이다.

퀵 정렬은 피벗을 기준으로 배열을 나누고, 나뉜 구간을 다시 정렬한다. 피벗이 균형 있게 선택되면 평균적으로 `O(nlogn)`의 시간 복잡도를 가지므로 큰 배열에서는 선택 정렬보다 빠른 경우가 많다.

| 구분 | 퀵 정렬 | 선택 정렬 |
| :--- | :--- | :--- |
| 정렬 방식 | 피벗을 기준으로 배열을 분할하며 정렬한다. | 가장 작은 값을 찾아 앞쪽부터 배치한다. |
| 최악의 경우 | `O(n^2)` | `O(n^2)` |
| 평균적인 경우 | `O(nlogn)` | `O(n^2)` |
| 최선의 경우 | `O(nlogn)` | `O(n^2)` |
| 특징 | 피벗 선택에 따라 성능이 달라진다. | 구현이 단순하지만 큰 배열에서는 비효율적이다. |

따라서 데이터가 많을수록 평균적으로는 퀵 정렬이 선택 정렬보다 효율적이다. 다만 현재 구현처럼 마지막 값을 피벗으로 사용하면 이미 정렬된 배열에서는 최악의 경우가 발생할 수 있다.

## 4. 버블 정렬의 작동 원리

버블 정렬(Bubble Sort)은 서로 이웃한 두 값을 비교하면서, 더 큰 값을 뒤쪽으로 보내는 정렬 알고리즘이다.

한 번 배열을 끝까지 훑으면 가장 큰 값이 배열의 맨 뒤로 이동한다. 이 모습이 물속의 거품이 위로 올라가는 것과 비슷해서 버블 정렬이라고 부른다.

작동 방식은 다음과 같다.

1. 첫 번째 값과 두 번째 값을 비교한다.
2. 앞의 값이 더 크면 두 값을 서로 바꾼다.
3. 두 번째 값과 세 번째 값을 비교한다.
4. 같은 방식으로 배열 끝까지 인접한 값을 비교하고 교환한다.
5. 한 번의 반복이 끝나면 가장 큰 값이 맨 뒤에 놓인다.
6. 이미 정렬된 맨 뒤 값을 제외하고 같은 과정을 반복한다.

예를 들어 다음 배열이 있다.

```text
[5, 3, 8, 4, 2]
```

### 1회전

`5`와 `3`을 비교한다. `5`가 더 크므로 위치를 바꾼다.

```text
[3, 5, 8, 4, 2]
```

`5`와 `8`을 비교한다. 순서가 맞으므로 그대로 둔다.

```text
[3, 5, 8, 4, 2]
```

`8`과 `4`를 비교한다. `8`이 더 크므로 위치를 바꾼다.

```text
[3, 5, 4, 8, 2]
```

`8`과 `2`를 비교한다. `8`이 더 크므로 위치를 바꾼다.

```text
[3, 5, 4, 2, 8]
```

1회전이 끝나면 가장 큰 값 `8`이 맨 뒤에 확정된다.

### 2회전

맨 뒤의 `8`은 이미 정렬되었으므로 제외하고, 앞쪽 구간 `[3, 5, 4, 2]`만 다시 비교한다.

```text
[3, 4, 2, 5, 8]
```

2회전이 끝나면 남은 구간에서 가장 큰 값 `5`가 제자리로 이동한다.

### 반복 후 결과

같은 과정을 계속 반복하면 최종적으로 배열은 다음처럼 정렬된다.

```text
[2, 3, 4, 5, 8]
```

코드에서는 다음 부분이 버블 정렬을 수행한다.

```java
public static void bubbleSort(int[] numbers) {
    for (int i = 0; i < numbers.length - 1; i++) {
        boolean swapped = false;

        for (int j = 0; j < numbers.length - 1 - i; j++) {
            if (numbers[j] > numbers[j + 1]) {
                swap(numbers, j, j + 1);
                swapped = true;
            }
        }

        if (!swapped) {
            return;
        }
    }
}
```

- 바깥 반복문은 전체 비교 과정을 몇 번 반복할지 정한다.
- 안쪽 반복문은 인접한 두 값을 비교하고, 순서가 잘못되어 있으면 교환한다.
- `numbers.length - 1 - i`를 사용해 이미 뒤쪽에 정렬된 값은 다시 비교하지 않는다.
- `swapped`는 이번 회전에서 교환이 있었는지 확인한다.
- 교환이 한 번도 없으면 이미 정렬된 상태이므로 바로 종료한다.

## 5. 버블 정렬의 시간 복잡도

버블 정렬은 인접한 값을 반복해서 비교하므로 일반적으로 중첩 반복문 구조를 가진다.

| 경우 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| 최악의 경우 | `O(n^2)` | 배열이 거꾸로 정렬되어 있어 거의 모든 비교마다 교환이 발생하는 경우이다. |
| 평균적인 경우 | `O(n^2)` | 일반적인 입력에서는 여러 회전에 걸쳐 인접한 값들을 계속 비교한다. |
| 최선의 경우 | `O(n)` | 이미 정렬되어 있고, `swapped` 검사로 첫 회전 후 바로 종료하는 경우이다. |

기본적인 버블 정렬은 최선의 경우에도 모든 회전을 반복하면 `O(n^2)`이 된다. 하지만 현재 `RoadToBiodome05_1.java` 구현은 `swapped`를 사용해 교환이 없을 때 바로 종료한다.

따라서 이미 정렬된 배열은 한 번만 훑고 끝낼 수 있으므로 최선의 경우 시간 복잡도는 다음과 같다.

```text
O(n)
```

하지만 배열이 많이 섞여 있거나 거꾸로 정렬되어 있으면 여러 번 반복해야 하므로 평균과 최악의 경우는 다음과 같다.

```text
O(n^2)
```

## 6. 퀵 정렬과 버블 정렬의 차이

퀵 정렬은 피벗을 기준으로 배열을 나누면서 정렬하고, 버블 정렬은 인접한 값을 계속 비교하면서 큰 값을 뒤로 보낸다.

| 구분 | 퀵 정렬 | 버블 정렬 |
| :--- | :--- | :--- |
| 정렬 방식 | 피벗을 기준으로 작은 값과 큰 값을 나눈다. | 인접한 두 값을 비교해 큰 값을 뒤쪽으로 보낸다. |
| 최악의 경우 | `O(n^2)` | `O(n^2)` |
| 평균적인 경우 | `O(nlogn)` | `O(n^2)` |
| 최선의 경우 | `O(nlogn)` | `O(n)` |
| 특징 | 평균적으로 빠르지만 피벗 선택에 영향을 받는다. | 원리는 단순하지만 큰 배열에서는 비효율적이다. |

작은 배열이나 학습용 예제에서는 버블 정렬이 이해하기 쉽다. 하지만 데이터가 많아질수록 평균적으로는 퀵 정렬이 버블 정렬보다 훨씬 효율적이다.
