## 1. 평균값과 중앙값의 차이점

### 평균값

평균값은 모든 값을 더한 뒤, 값의 개수로 나눈 값이다.

예를 들어 다음 배열이 있을 때:

```text
[5, 3, 8, 9, 4]
```

모든 값을 더하면 다음과 같다.

```text
5 + 3 + 8 + 9 + 4 = 29
```

값은 총 5개이므로 평균값은 다음과 같다.

```text
29 / 5 = 5.8
```

따라서 평균값은 `5.8`이다.

### 중앙값

중앙값은 값을 오름차순으로 정렬했을 때 가운데에 위치한 값이다.

```text
[5, 3, 8, 9, 4]
```

위 배열을 오름차순으로 정렬하면 다음과 같다.

```text
[3, 4, 5, 8, 9]
```

가운데 값은 `5`이므로 중앙값은 `5`이다.

### 짝수 개일 때 중앙값

배열의 길이가 짝수이면 정확히 가운데 하나의 값이 없다. 이 경우 가운데에 있는 두 값의 평균을 중앙값으로 사용한다.

예를 들어:

```text
[7, 2, 10, 6]
```

정렬하면 다음과 같다.

```text
[2, 6, 7, 10]
```

가운데 두 값은 `6`과 `7`이다. 따라서 중앙값은 다음과 같다.

```text
(6 + 7) / 2 = 6.5
```

### 평균값과 중앙값의 차이

- 평균값은 모든 값을 계산에 사용한다.
- 중앙값은 정렬 후 가운데 위치의 값을 사용한다.
- 평균값은 너무 크거나 작은 값의 영향을 많이 받는다.
- 중앙값은 극단적인 값의 영향을 평균값보다 덜 받는다.

예를 들어:

```text
[1, 2, 3, 4, 100]
```

평균값은 다음과 같다.

```text
(1 + 2 + 3 + 4 + 100) / 5 = 22
```

하지만 정렬된 배열의 가운데 값은 `3`이므로 중앙값은 `3`이다.

이처럼 `100`처럼 매우 큰 값이 있으면 평균값은 크게 변하지만, 중앙값은 상대적으로 영향을 덜 받는다.

## 2. 선택 정렬의 작동 원리

선택 정렬(Selection Sort)은 배열에서 가장 작은 값을 찾아 앞쪽부터 차례대로 배치하는 정렬 알고리즘이다.

작동 방식은 다음과 같다.

1. 배열 전체에서 가장 작은 값을 찾는다.
2. 그 값을 배열의 첫 번째 위치와 바꾼다.
3. 두 번째 위치부터 다시 가장 작은 값을 찾는다.
4. 그 값을 두 번째 위치와 바꾼다.
5. 이 과정을 배열 끝까지 반복한다.

예를 들어 다음 배열이 있다.

```text
[5, 3, 8, 9, 4]
```

### 1회전

전체 배열에서 가장 작은 값은 `3`이다. `3`을 첫 번째 위치의 `5`와 바꾼다.

```text
[3, 5, 8, 9, 4]
```

### 2회전

두 번째 위치부터 끝까지 보면 `[5, 8, 9, 4]`이다. 여기서 가장 작은 값은 `4`이다. `4`를 두 번째 위치의 `5`와 바꾼다.

```text
[3, 4, 8, 9, 5]
```

### 3회전

세 번째 위치부터 끝까지 보면 `[8, 9, 5]`이다. 가장 작은 값은 `5`이다. `5`를 세 번째 위치의 `8`과 바꾼다.

```text
[3, 4, 5, 9, 8]
```

### 4회전

네 번째 위치부터 끝까지 보면 `[9, 8]`이다. 가장 작은 값은 `8`이다. `8`을 네 번째 위치의 `9`와 바꾼다.

```text
[3, 4, 5, 8, 9]
```

최종적으로 오름차순 정렬이 완료된다.

```text
[3, 4, 5, 8, 9]
```

코드에서는 다음 부분이 선택 정렬을 수행한다.

```java
public static void selectionSort(int[] numbers) {
    for (int i = 0; i < numbers.length - 1; i++) {
        int minIndex = i;

        for (int j = i + 1; j < numbers.length; j++) {
            if (numbers[j] < numbers[minIndex]) {
                minIndex = j;
            }
        }

        int temp = numbers[i];
        numbers[i] = numbers[minIndex];
        numbers[minIndex] = temp;
    }
}
```

- 바깥 반복문은 정렬할 위치를 하나씩 이동한다.
- 안쪽 반복문은 아직 정렬되지 않은 구간에서 가장 작은 값을 찾는다.
- 가장 작은 값의 위치인 `minIndex`를 찾으면 현재 위치 `i`와 값을 교환한다.

## 3. 선택 정렬의 시간 복잡도

선택 정렬은 배열이 이미 정렬되어 있든, 거꾸로 정렬되어 있든, 무작위로 섞여 있든 항상 남은 구간 전체를 확인하면서 가장 작은 값을 찾는다.

따라서 최악, 평균, 최선의 경우 모두 시간 복잡도가 같다.

| 경우 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| 최악의 경우 | `O(n^2)` | 배열이 어떤 상태여도 중첩 반복문으로 비교를 수행한다. |
| 평균적인 경우 | `O(n^2)` | 일반적인 입력에서도 각 위치마다 남은 값들을 모두 비교한다. |
| 최선의 경우 | `O(n^2)` | 이미 정렬된 배열이어도 가장 작은 값을 찾기 위해 비교를 계속 수행한다. |

예를 들어 배열의 길이가 `n`일 때 선택 정렬의 비교 횟수는 대략 다음과 같다.

```text
(n - 1) + (n - 2) + (n - 3) + ... + 1
```

이 값은 다음과 같이 정리할 수 있다.

```text
n(n - 1) / 2
```

Big O 표기법에서는 상수와 낮은 차수 항을 생략하므로 선택 정렬의 시간 복잡도는 다음과 같다.

```text
O(n^2)
```

## 4. 구현한 프로그램에서 평균값과 중앙값을 구하는 방식

### 평균값 계산

```java
public static double calculateAverage(int[] numbers) {
    int sum = 0;
    for (int i = 0; i < numbers.length; i++) {
        sum += numbers[i];
    }
    return (double) sum / numbers.length;
}
```

배열의 모든 값을 더한 뒤 배열 길이로 나눈다. 배열 전체를 한 번 순회하므로 시간 복잡도는 `O(n)`이다.

### 중앙값 계산

```java
public static double calculateMedian(int[] sortedNumbers) {
    int length = sortedNumbers.length;
    int middle = length / 2;

    if (length % 2 == 1) {
        return sortedNumbers[middle];
    }

    return (sortedNumbers[middle - 1] + sortedNumbers[middle]) / 2.0;
}
```

중앙값은 정렬된 배열을 기준으로 계산한다.

- 배열 길이가 홀수이면 가운데 인덱스의 값을 반환한다.
- 배열 길이가 짝수이면 가운데 두 값의 평균을 반환한다.

중앙값 계산 자체는 배열의 특정 인덱스에 접근만 하므로 `O(1)`이다. 하지만 중앙값을 구하려면 먼저 선택 정렬을 해야 하므로 전체 과정은 선택 정렬의 시간 복잡도인 `O(n^2)`의 영향을 가장 많이 받는다.
