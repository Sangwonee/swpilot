## 개념 확인 질문 정리

### 1. BFS 알고리즘은 무엇이며 어떻게 동작하는가?

BFS(Breadth First Search)는 너비 우선 탐색이라는 뜻으로, 시작점에서 가까운 위치부터 순서대로 탐색하는 알고리즘이다.

이번 문제에서는 시작점 `(0, 0)`을 기준으로 다음 순서로 동굴을 탐색한다.

```text
거리 0인 칸 탐색
→ 거리 1인 칸들을 모두 탐색
→ 거리 2인 칸들을 모두 탐색
→ 거리 3인 칸들을 모두 탐색
```

BFS는 먼저 발견한 위치를 먼저 처리하기 위해 FIFO 방식의 Queue를 사용한다.

1. 시작 좌표를 Queue에 넣고 방문 처리한다.
2. Queue의 맨 앞 좌표를 꺼낸다.
3. 현재 좌표의 상하좌우를 확인한다.
4. 동굴 안에 있고 값이 `1`이며 방문하지 않은 좌표를 Queue에 넣는다.
5. 새 좌표의 거리를 현재 거리보다 1 크게 저장한다.
6. Queue가 비거나 출구에 도착할 때까지 반복한다.

```text
Queue에 좌표 추가: offer()
Queue에서 좌표 제거: poll()
```

각 칸으로 이동하는 비용이 모두 1일 때 BFS는 가까운 거리의 칸부터 탐색한다. 따라서 출구를 처음 방문했을 때 저장된 거리가 최단 거리이다.

`visited` 배열은 이미 방문한 칸을 다시 Queue에 넣지 않도록 막는다. 방문 여부를 기록하지 않으면 같은 칸을 반복해서 이동하며 불필요한 탐색이 발생할 수 있다.

동굴이 `N x M` 크기라면 각 칸을 최대 한 번씩 방문하므로 시간 복잡도는 `O(N x M)`이고, Queue와 방문 배열 등에 필요한 공간 복잡도도 `O(N x M)`이다.

### 2. Queue를 직접 구현하는 것과 Java 기본 Queue를 사용하는 것은 어떤 차이가 있는가?

Queue를 직접 구현하면 배열과 `front`, `rear`, `size`를 이용해 데이터가 들어오고 나가는 과정을 직접 관리해야 한다.

```text
enqueue: rear를 이동한 후 값 저장
dequeue: front의 값을 꺼낸 후 front 이동
isEmpty: 현재 저장된 데이터 개수 확인
```

직접 구현하는 방식은 Queue의 FIFO 원리, 원형 배열, 빈 상태와 가득 찬 상태를 이해하는 데 도움이 된다. 하지만 배열 크기 확장, 인덱스 처리, 예외 상황까지 직접 작성해야 하므로 코드가 길어지고 오류가 발생할 가능성이 커진다.

Java 기본 Queue를 사용하면 이미 구현되고 검증된 기능을 바로 사용할 수 있다.

```java
Queue<int[]> positions = new ArrayDeque<>();

positions.offer(new int[]{0, 0}); // 뒤에 좌표 추가
int[] position = positions.poll(); // 앞의 좌표 제거 후 반환
positions.isEmpty();               // Queue가 비었는지 확인
```

기본 Queue는 저장할 데이터가 늘어날 때 내부 공간을 직접 관리해 주므로 기능을 빠르게 구현할 수 있고, 사용자가 `front`와 `rear`를 직접 관리할 필요가 없다. 또한 `Queue<int[]>`처럼 제네릭을 이용해 좌표 배열 등의 여러 자료형을 저장할 수 있어 확장하기 쉽다.

정리하면 Queue의 작동 원리를 학습하는 것이 목적이라면 직접 구현하는 방식이 적합하고, BFS와 같은 실제 기능을 빠르고 안정적으로 구현하는 것이 목적이라면 Java 기본 Queue를 사용하는 것이 적합하다.
