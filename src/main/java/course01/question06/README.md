# [개념 확인] 자료형의 차이와 문자 비교 원리 (Question 06)

## 1. 문자를 `==` 연산자로 비교할 수 있는가?
* **답변**: **네, 직접 비교할 수 있습니다.**
* **이유**: 자바에서 단일 문자를 나타내는 `char` 타입은 내부적으로 **UTF-16 코드의 정수값**으로 저장되는 기본 자료형(Primitive Type) 입니다. 따라서 `==` 연산자를 사용해 메모리 속 실제 정수값 자체를 직접 비교(`char1 == char2`)할 수 있으며, 값이 같으면 올바르게 참(True)을 반환합니다.
* **주의**: 참조 자료형인 문자열 객체(`String`)끼리는 `==`로 비교하면 실제 글자 값이 아닌 메모리 주소를 비교하게 되므로 불가능하지만, `charAt(i)` 등으로 추출한 개별 문자(`char`)는 `==` 비교가 정상 작동합니다.

## 2. Java 기본 자료형과 참조 자료형의 개념 차이
* **기본 자료형 (Primitive Type)**
  * 비객체 타입으로, 실제 데이터 값을 변수 자체에 직접 저장합니다.
  * 예: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean` (총 8가지)
* **참조 자료형 (Reference Type)**
  * 실제 객체가 저장된 주소(참조값)를 변수에 저장합니다.
  * 예: `String`, `Array`, `Class`, `Interface` 등 (기본 자료형을 제외한 모든 타입)

---

## 3. 저장 값 및 저장 메모리 영역의 차이

| 구분 | 기본 자료형 (Primitive Type) | 참조 자료형 (Reference Type) |
| :--- | :--- | :--- |
| **저장되는 값** | **실제 데이터 값** (정수, 실수, 문자, 논리값 등) | 실제 객체가 저장된 **메모리 주소값 (참조값)** |
| **메모리 영역** | **스택(Stack) 영역**에 변수와 실제 값이 함께 생성되어 직접 저장됩니다. | **스택(Stack) 영역**에는 객체의 **주소값**만 저장되고, 실제 데이터(객체)는 **힙(Heap) 영역**에 생성됩니다. |
| **비교 연산 (`==`)** | **실제 저장된 값**이 같은지 비교합니다. | 가리키고 있는 **메모리 주소**가 같은지 비교합니다. |
| **메모리 수명** | 메소드 실행 종료 시 스택에서 즉시 사라집니다. | 더 이상 주소를 참조하지 않을 때 가비지 컬렉터(GC)에 의해 힙에서 해제됩니다. |

### 💡 메모리 구조 예시
* `int age = 20;` ➔ 스택의 `age`라는 공간에 실제 정수 `20`이 저장됨.
* `String name = "Biodome";` ➔ 실제 `"Biodome"` 글자 객체는 힙(Heap) 에 할당되고, 스택의 `name` 변수 공간에는 힙 객체의 **시작 주소값**(예: `0x7f30`)이 저장됨.
