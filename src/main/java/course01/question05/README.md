# [개념 확인] Java 연산자 우선순위 & 10진수-2진수 상호 변환

## 1. Java 연산자 종류별 우선순위 (높은 순 -> 낮은 순)
자바 수식 계산 시 연산자는 아래의 정해진 우선순위 순서대로 결합하고 동작합니다.

1. **단항 연산자 (Unary)**: `+`, `-` (부호), `~` (비트 반전), `!` (논리 부정)
2. **산술 연산자 (Arithmetic)**: 곱셈 계열(`*`, `/`, `%`) -> 덧셈 계열(`+`, `-`)
3. **시프트 연산자 (Shift)**: `<<`, `>>`, `>>>` (비트를 좌/우로 이동)
4. **비교 연산자 (Comparison)**: 대소 비교(`<`, `>`, `<=`, `>=`) -> 등가 비교(`==`, `!=`)
5. **비트 연산자 (Bitwise)**: AND(`&`) -> XOR(`^`) -> OR(`|`)
6. **논리 연산자 (Logical)**: AND(`&&`) -> OR(`||`)
7. **대입 연산자 (Assignment)**: `=`, `+=`, `-=`, `&=` 등 (가장 낮음)

* **핵심 요약**: 단항 ➔ 산술 ➔ 시프트 ➔ 비교 ➔ 비트 ➔ 논리 ➔ 대입
* **주의점**: 비교 연산자(`==`)가 비트 연산자(`&`, `^`, `|`)보다 우선순위가 높기 때문에, 괄호가 없으면 비교 연산이 비트 연산보다 먼저 수행됩니다.

## 2. 10진수와 2진수 상호 변환 방법

### A. 2진수 ➔ 10진수 변환
* **방법**: 2진수의 각 자릿수에 해당하는 자리값($2^0, 2^1, 2^2, 2^3, \dots$)을 각 자리 숫자($0$ 또는 $1$)에 곱한 후 모두 더합니다.
* **예시**: `0001 0101`을 10진수로 변환
  * 자리값 계산:
    * $2^4$ 자리: $1 \times 16 = 16$
    * $2^2$ 자리: $1 \times 4 = 4$
    * $2^0$ 자리: $1 \times 1 = 1$
  * 합산 결과: $16 + 4 + 1 = \mathbf{21}$

### B. 10진수 ➔ 2진수 변환
* **방법**: 10진수 정수를 몫이 0이 될 때까지 2로 계속 나누면서 발생하는 나머지를 기록한 뒤, 마지막에 나온 나머지부터 거꾸로(역순) 읽어 올라갑니다.
* **예시**: 10진수 `28`을 2진수로 변환
  * $28 \div 2 = 14 \quad \dots \quad \text{나머지 } 0$ (우측 끝 자리)
  * $14 \div 2 = 7 \quad \dots \quad \text{나머지 } 0$
  * $7 \div 2 = 3 \quad \dots \quad \text{나머지 } 1$
  * $3 \div 2 = 1 \quad \dots \quad \text{나머지 } 1$
  * $1 \div 2 = 0 \quad \dots \quad \text{나머지 } 1$ (종료)
  * 나머지를 아래에서 위로 읽어 완성된 이진수: `11100` ➔ 8비트 규격 맞춤: **`0001 1100`**
