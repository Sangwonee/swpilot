수행과제

[최종 결과물]
사용자로부터 입력 받은 온도, 습도, 산소 농도 값의 범위를 확인하고, 안정 범위에 있는지 판단하는 Java 프로그램을 구현한다.

[과제 목표]
조건문의 개념을 이해하고, Java에서 조건문을 구현하는 방법을 숙지한다.

[기능 요구사항]

사용자로부터 온도, 습도, 산소 농도 3가지 값을 입력 받는다.

사용자가 값을 모두 입력하면 각 항목별로 적절한 범위를 검토한다.

모든 값이 안정 범위에 있으면 “생명의 나무는 안정적인 상태입니다” 문구를 출력한다.

안정 범위를 벗어나는 값이 있으면 “XX값이 정상 범위를 벗어났습니다. 확인이 필요합니다.” 문구를 출력한다.

[구현 지침]

파일의 이름은 HelloBiodome04.java 이다.

온도, 습도, 산소 농도 값을 저장할 각 변수를 생성한다.

안정한 상태일 때의 온도/습도/산소 농도의 각 범위 아래와 같다.

온도 : 10° 이상 27.5°C 미만

습도 : 40% 초과 60% 미만

산소 농도 : 19.5% 이상 23.5% 이하

사용자가 입력한 환경 값의 범위를 검토하기 위해 조건문을 활용해 boolean값을 반환하는 메서드를 구현한다.

main 함수에서 검토 결과에 따른 결과(안전/확인필요) 문구를 출력한다.

정상 범위를 벗어나는 요소가 무엇인지 안내해야한다.

한번의 입력에서 정상을 벗어날 수 있는 요소는 최대 1개이다.
(온도와 습도가 동시에 정상 범위를 벗어날 수 없다)

예외 상황이 발생하는 경우 상황에 맞는 안내메시지를 출력해야한다.

숫자값 3개가 모두 입력되지 않은 경우

입력된 값에 문자가 포함되는 경우

[결과 예시]

25.5 65.0 21.0
-> 생명의 나무는 안정적인 상태입니다 :)

-27 55 20
→ 온도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.

3.2 55.1 aa
-> 입력된 값이 올바르지 않습니다. [온도][습도][산소농도] 순서대로 숫자 값을 입력해주세요

제약사항

사용자 입력은 Command Line Arguments 를 이용해 받는다.
Scanner , BufferedReader 를 사용할 수 없다.



보너스 과제

나무가 안정적인 경우(세가지 환경 정보가 모두 정상 범위 일때), 나무의 건강 지수 H 값을 출력한다.

계산된 건강지수 H 값을 소수점 셋째 자리에서 반올림하여 출력한다.

Math 패키지 사용할 수 없다.

for 와 while 반복문을 사용할 수 없다

23.1 60.0 23.0
-> 생명의 나무는 안정적인 상태입니다. 건강지수는 9.23입니다.

평가기준

[기본 확인]

파일명이 HelloBiodome04.java 인지 확인한다.

에러나 예기치 않은 종료없이 프로그램이 실행되어야한다.

[프로그램 구현 확인]

온도, 습도, 산소 농도 값을 저장할 각 변수를 적절한 숫자 데이터 타입으로 생성했는지확인한다.

사용자가 입력한 값의 범위를 검토하기 위해 조건문을 사용한 메서드를 구현했는지 확인한다.(if-else 또는 switch)

안정 범위의 값을 검토하기 위한 조건문에 이상, 이하, 초과, 미만에 적합한 논리 연산자를 사용했는지 확인한다.

[프로그램 동작 확인]

사용자가 입력한 값의 검토 결과(안전/확인 필요)를 올바르게 출력하는지 확인한다.

정상 범위를 벗어나는 경우, 어떤 요소가 벗어나는지 안내해야한다.

25.5 65.0 21.0
-> 생명의 나무는 안정적인 상태입니다

-27 55 20
-> 온도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.

문자 값을 입력 받을 시 안내 메시지를 출력해야한다.

3.2 55.1 aa
-> 입력된 값이 올바르지 않습니다. [온도][습도][산소농도] 순서대로 숫자 값을 입력해주세요

환경 값 3개를 모두 입력 받지 못한 경우 안내 메시지를 출력해야한다.

3.2 55.1
-> 입력되지 않은 값이 있습니다. [온도][습도][산소농도] 순서대로 숫자 값을 입력해주세요

[개념 확인 질문]

각 요소의 범위를 체크하기 위해 사용한 조건문의 작동 흐름과 순서를 질문한다.

Java에서 조건문을 구현할 수 있는 if-else 구문과 switch 구문의 장단점을 질문한다.

본 과제 적용된 조건문을 다른 방식(만일 if-else 였다면 switch로, switch 였다면 if-else)으로 변경하여 에러없이 작동하도록 구현 요청한다.
